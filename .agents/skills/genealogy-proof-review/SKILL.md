# Genealogy Proof Review

Use this skill before any staged material is promoted into canonical wiki folders.

## Review Workflow

1. Read the staged draft and its referenced converted file, chunk, page, or source packet.
2. Check that literal support exists and matches the proposed claim or conclusion.
3. Confirm transcription, translation, interpretation, and uncertainty are separated.
4. Check that source reliability, conversion confidence, and claim confidence are separated.
5. Look for missing conflicts, identity risks, chronology problems, or unsupported relationship jumps.
6. Return a recommendation: `promote`, `revise`, `hold`, or `reject`.
7. Write review notes under `research/_staging/reviews/` only when the user asks for saved review artifacts.

## Review Criteria

- Source path, converted file, and chunk/page reference are present.
- Each claim is atomic and not bundled with unrelated facts.
- Status and confidence are conservative.
- Source reliability is evaluated for the specific claim, not blindly inherited from the source title.
- Conversion confidence is checked against any `research/_conversion-review/` notes.
- Relationships are evidence-backed and not inferred from proximity alone.
- Same-person candidates remain candidates unless identity evidence is explicit.
- Conflicts and uncertainty remain visible.
- Canonical target pages are identified for promotion.

## Promotion Boundary

This skill reviews; it does not promote. Only the `wiki_promoter` role, or explicit human direction, should move reviewed drafts into canonical `wiki/` folders.

## Output

Lead with findings that block promotion. Then list drafts ready to promote and drafts that should remain staged. Include file paths and page/chunk references wherever possible.
